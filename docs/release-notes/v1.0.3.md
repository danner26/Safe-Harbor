# v1.0.3 — operator-experience polish

## Breaking

`TRUST_PROXY_HEADERS` now defaults to `0`, changed from the previous default of `1`.

Migration: operators running behind ANY trusted reverse proxy or tunnel that strips or controls client-supplied `X-Forwarded-*` headers and blocks direct app access, including Cloudflare Tunnel, nginx, traefik, Caddy, or similar deployments, MUST set `TRUST_PROXY_HEADERS=1` explicitly in their environment. Direct-expose deployments need no change.

## Operator-experience hardening

- `_safe_float_env` now logs a WARNING through Python standard-library `logging` when an environment variable is set but unparseable, most visibly for `SENTRY_TRACES_SAMPLE_RATE`.
- `safeharbor backup` now validates `BACKUP_RETENTION_DAILY` and `BACKUP_RETENTION_WEEKLY` before creating the tarball, rejecting non-integer or negative values with a clear `BadParameter` error.
- `.env.example` includes a prominent comment explaining the new `TRUST_PROXY_HEADERS` default and when to flip it.

## CI / release pipeline

Multi-arch container builds now run on native amd64 and arm64 runners. CI workflows now cache uv, mypy, and ruff state.

## No schema changes

No migrations are required for this release; `flask db upgrade` is a no-op against a v1.0.2 database.
